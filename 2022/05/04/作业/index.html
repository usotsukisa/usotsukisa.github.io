    虚拟机的配置

    配置虚拟光驱

    安装好虚拟机之后，我们需要配置虚拟光驱，来作为本地的RPM源。为后面创建YUM仓库做准备
    首先打开VMware上方菜单栏的虚拟机选项，然后在虚拟机选项中选择CD/DVD(IDE)之后点击设置。
    如图设置
    确认虚拟光驱接入设备后，我们开始进行光驱挂载操作

    mount挂载操作

    mkdir /dvd //创建挂载点目录
mount /dev/cdrom /dvd //挂载命令 /挂载点 /挂载目录
ls /dvd //用ls命令查看是否挂载成功，如出现文件目录则为成功

    YUM仓库的配置

    首先先书写YUM仓库的配置文件
    rm -rf /etc/yum.repos.d/* //清空原来的配置文件
    vim /etc/yum.repos.d/neme.repo //创建新配置文件
    接下来是vim内书写格式

    至此，YUM仓库的配置已经完成。

    安装apeche(httpd)和vsftpd(FTP服务器)

    yum -y install httpd //安装httpd
yum -y install vsftpd //安装vsftpd
systemctl start name //启动某服务
systemctl enable name //设置某服务自动启动

    如需测试httpd服务是否正常启动，键入命令firefox 网卡IP地址或者是本机回环地址。
    网页默认存放在/var/www/html/下，文件名为index.html。

    firewall(防火墙)的配置

    用yum -y install安装firewell，有firewall-cmd(命令行工具) firewall-config(图形化工具)两种
    firewall拥有四个区域规则，分别为public(仅允许使用个别服务) trusted(允许所有访问)
    block(阻塞所有请求 明确拒绝) drop(丢弃所有请求 只丢弃不拒绝 节约资源)。
    firewall-cmd --get-default-zone //查看防火墙默认区域
    修改默认区域
    firewall-cmd --set-default-zone=public trusted block drop
    查看防火墙区域策略
    firewall-cmd --zone=public trusted block drop --list-all
    添加区域协议
    firewall-cmd --zone=public trusted block drop --add-service=http ftp pop3
    将规则定义为永久生效(生效前需重启服务)
    firewall-cmd --permanent
    重启服务
    firewall-cmd --reload
    在区域中添加ip地址
    firewall-cmd --set-default-zone=public trusted block drop --add-source=127.0.0.1
    端口映射命令
    firewall-cmd --zone=public --add-forword-port=port=4000:proto=tcp:toport=80 //port:端口 proto:协议 toport:映射端口
    至此，firewall命令已经基本列出完成。接下来我们转进安装配置个人博客框架Hexo

    个人博客hexo的配置

    安装Node.js和git

    wget https://mirrors.tuna.tsinghua.edu.cn/nodejs-release/v16.15.0/node-v16.15.0-linux-x64.tar.gz //我们使用wget命令下载node.js的源码包(wget是一个网络下载工具，支持http和ftp下载方式)
tar -xzvf node-v16.15.0-linux-x64.tar.gz //解压下载好的源码包
mv node-v10.9.0-linux-x64 nodejs //修改名字方便管理
ln -s /nodejs/bin/node /usr/local/bin/node //设置软链接(类似于快捷方式)
ln -s /nodejs/bin/npm /usr/local/bin/npm
npm install hexo-deployer-git --save //安装git(git是分布式版本控制系统) npm是node.js的包管理系统 类似于rpm 可以自己上传包和命令行程序 也可以使用他人上传的

    安装Hexo框架

    npm install hexo-cli -g //安装hexo框架
ln -s /soft/hexo/nodejs/lib/node_modules/hexo-cli/bin/hexo /usr/local/bin/hexo //还是设置软链接
//执行以下命令，Hexo将会自动新建所需文件与文件夹(此部分来自于官方文档)
hexo init 
cd
npm install 
npm install hexo-server --save //安装关于Hexo服务器启动的插件

    这个时候可以启动服务器, 本地查看打开 http://localhost:4000 查看效果效果, 如果不指定端口，默认为4000
    hexo server
    至此我们搭建Hexo的流程就结束了，接下来我们将他部署到github page上，就可以拥有一个简易的个人博客了。

    将博客部署至github page

    此部分参考了https://juejin.cn/post/7014675289728876574，由于过程比较繁琐，这里不再赘述。

    大致思路：

    源码文件 Push 到 GitHub Blog 私有仓库 master 分支
    Github Action 在 Blog 私有仓库 master 分支更新后，自动构建生成站点文件
    Github Action 将生成的静态文件推送到 username.github.io 仓库 master 分支
    云服务器从 GitHub Pagemaster 分支拉取更新
    也就是说，整个部署过程只需要将写好源码文件 Push 到 GitHub 上存放源码仓库的 master 分支，后面的操作交给 Github Action 处理即可。
